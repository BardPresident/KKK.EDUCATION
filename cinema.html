üöÄ How To Use This (3 Steps)

STEP 1: Get the script

Download generate-manifest.js from wherever you found this cinema page.

Put it in the same folder as cinema.html.

STEP 2: Configure it (ONE LINE TO CHANGE)

Open generate-manifest.js in any text editor (Notepad, TextEdit, VS Code, whatever).

Find line 12. Change it to point to your videos:

// Line 12 - CHANGE THIS:
const ARCHIVE_DIR = '/path/to/your/videos';

// Examples:
// Mac:     const ARCHIVE_DIR = '/Users/yourname/Videos';
// Windows: const ARCHIVE_DIR = 'C:\\Users\\YourName\\Videos';
// Server:  const ARCHIVE_DIR = '/var/www/html/videos';
// Local:   const ARCHIVE_DIR = './my-videos';

That's it. One line. Save the file.

STEP 3: Run it

Open terminal (Mac/Linux) or Command Prompt (Windows).

Navigate to where you saved the script:

cd /path/to/your/cinema/folder

Run the script:

node generate-manifest.js

This creates manifest.json with your entire file tree.

Now open cinema.html in your browser. Your cinema is live.

üí° IMPORTANT: WEB_PREFIX

In generate-manifest.js, there's a line that says:

const WEB_PREFIX = '/videos';

This tells the script how browsers will access your files.

If your videos are at https://yoursite.com/videos/... then use '/videos'

If they're in the root https://yoursite.com/file.mp4 then use '' (empty)

‚ùì Troubleshooting

Q: "Manifest not found"

A: You need to run node generate-manifest.js first.

Q: "Directory not found" when running script

A: Check line 12 in generate-manifest.js - make sure the path is correct.

Q: Videos won't stream

A: Check WEB_PREFIX in the script. Make sure it matches how browsers access your files.

Q: Nothing shows up

A: Press F12 in your browser. Look at the Console tab. It'll tell you what's wrong.
